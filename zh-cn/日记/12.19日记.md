关于temp指针：

我们要解决的核心问题是：**为什么修改 `curr.next` 之前，必须先用 `temp` 把后面的节点存起来？**

假设我们现在的链表是：`A -> B -> C`。
我们要把 `A` 的箭头反过来指向 `null`（或者前一个节点），即**反转**操作。

---

### 场景演示：如果不使用 temp（灾难现场）

#### 1. 初始状态
内存里有三个节点，地址分别是 100, 200, 300。
`curr` 指针指向 A。

```text
       curr
        |
        v
    [ 节点 A ] ----------> [ 节点 B ] ----------> [ 节点 C ]
    (地址:100)             (地址:200)             (地址:300)
```

此时，你只能通过 A 找到 B。如果不经过 A，你根本不知道 B 在哪里（地址 200 是未知的）。

#### 2. 执行错误操作
你直接写了代码：`curr.next = null;` （或者是 `prev`）
意思是：把 A 的指针切断，让它不再指向 B，而是指向空。

**操作发生后的瞬间：**

```text
       curr
        |
        v
    [ 节点 A ] --//--> null      [ 节点 B ] ----------> [ 节点 C ]
    (地址:100)                   (地址:200)             (地址:300)
                                    ^
                                    |
                                 没人指向它了！
                                 (彻底失联)
```

#### 3. 后果
*   现在你想处理 B。
*   你问 `curr` (A)："你的下一个是谁？" -> A 回答："我指向 null。"
*   你问系统："B 在哪里？" -> 系统回答："刚才只有 A 知道 B 在哪，现在 A 不指向 B 了，我也不知道 B 在哪。"
*   **结果**：链表从 B 开始往后所有的节点（B, C...）全部丢失。在 Java 中，垃圾回收器（GC）会认为这块内存没人用了，直接回收销毁。

---

### 场景演示：正确操作（使用 temp 救场）

让我们回到初始状态，看看加了 `temp` 之后发生了什么。

#### 1. 初始状态

```text
       curr
        |
        v
    [ 节点 A ] ----------> [ 节点 B ] ----------> [ 节点 C ]
```

#### 2. 第一步：保存现场 (定义 temp)
执行代码：`ListNode temp = curr.next;`
意思是：在切断 A 和 B 的联系之前，先新建一个指针 `temp`，让它也指向 B。

```text
       curr                   temp
        |                      |
        v                      v
    [ 节点 A ] ----------> [ 节点 B ] ----------> [ 节点 C ]
```
**重点**：现在有两个箭头指向 B。一个是 A 的 `next`，一个是我们的变量 `temp`。

#### 3. 第二步：安全切断 (修改指向)
执行代码：`curr.next = null;` (或者是 `prev`)
意思是：把 A 的指针切断，指向别处。

```text
       curr                   temp
        |                      |
        v                      v
    [ 节点 A ] --//--> null  [ 节点 B ] ----------> [ 节点 C ]
```

**重点**：
*   A 和 B 之间的线断了。
*   **但是！** B 并没有丢失，因为 `temp` 手里还紧紧抓着 B 的地址。

#### 4. 第三步：继续前进
执行代码：`curr = temp;`
意思是：我们处理完 A 了，现在通过 `temp` 这个桥梁，走到 B 那里去继续工作。

```text
                             curr (移过来了)
                              |
                              v
    [ 节点 A ]             [ 节点 B ] ----------> [ 节点 C ]
```
